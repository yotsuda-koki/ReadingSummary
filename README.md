# ReadingSummary

読書の習慣化と、読んだ内容の定着をサポートするための読書管理・要約ログアプリです。
「本を読んでも内容を忘れてしまう」「読書がなかなか続かない」といった課題を、ログの記録、要約の作成、データの可視化によって解決します。

## 解決する課題
- **内容が定着しない:** 章単位や本全体でのMarkdown形式による要約保存。
- **継続が難しい:** 連続読書日数（Streak）や月間読書時間の可視化によるモチベーション維持。

## 主な機能
- **本の管理:** タイトル、著者、言語、難易度、状態（未読/読書中/読了）の登録・編集。
- **読書セッション記録:** 日付、読書時間（分）、進捗（ページ数）、メモの記録。
- **要約保存:** Markdownエディタを使用した、章ごとまたは本全体単位での要約作成。
- **ダッシュボード:** 連続読書日数、今月の総読書時間、作成した要約数、読了本数の表示。
- **フィルタ・検索:** 本の読書状態による絞り込みや、タイトルでの検索機能。

## 画面構成
| 画面/URL | 目的 |
| :--- | :--- |
| `/login` | 認証（JWTによるログイン） |
| `/dashboard` | 読書データの可視化（KPI表示） |
| `/books` | 登録した本の一覧表示・検索 |
| `/books/new` | 新しい本の登録 |
| `/books/:id` | 本の詳細、進捗更新、セッション履歴、要約一覧 |
| `/books/:id/summary` | Markdownエディタによる要約の編集・保存 |

## 技術スタック（設計）

### Backend
- **Java / Spring Boot**
  - Spring Security (認証・認可)
  - Spring Data JPA (データアクセス)
  - Maven

### Frontend
- **React** (メインUI / 読書ダッシュボード)
- **Vue.js** (管理機能 / 統計ビュー)

### DB
- **PostgreSQL** (リレーショナルデータベース)

## 主要なAPIエンドポイント
- `POST /api/books` - 本の登録
- `GET /api/books` - 本の一覧取得
- `POST /api/books/{id}/sessions` - 読書セッションの追加
- `POST /api/books/{id}/summaries` - 要約の追加・更新
- `GET /api/dashboard` - 集計データ（KPI）の取得
